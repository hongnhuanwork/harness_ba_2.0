# Epic Template

## Purpose
Outlines high-level backlog groupings (Epics) that encompass related user stories.

## Traceability
Traces back to: BRD Business Objectives. Traces forward to: User Stories, Release Notes.

## Metadata
- Epic ID: [EP-01]
- Title: [Epic Name]
- Epic Owner: [Name]
- Status: [Backlog/Active/Completed]

## Required Sections
## 1. Epic Description & Objectives
Describe the scope and objective of this feature cluster.

## 2. Value Proposition
Detail the business value and target KPIs addressed by this Epic.

## 3. Child User Stories
- [US-01]: [Story Title] ([Points])
- [US-02]: [Story Title] ([Points])

## Optional Sections
## 4. Scope Exclusions
Explicitly detail items related to this feature area that are out of scope.

## Completion Checklist
- [ ] The Epic aligns to a business goal.
- [ ] All child user stories are linked.
- [ ] Dependencies are documented.
