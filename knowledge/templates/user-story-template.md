# User Story Template

## Purpose
Formulates developer-ready user stories with structured Gherkin acceptance criteria.

## Traceability
Traces back to: FRD, Use Case. Traces forward to: QA Test Cases, Code commits.

## Metadata
- Story ID: [US-01]
- Epic Link: [EP-01]
- Priority: [High/Medium/Low]
- Story Points: [Points]

## Required Sections
## 1. User Story Statement
**As a** [User Persona]
**I want to** [Perform Action]
**So that** [Deliver Business Value]

## 2. Acceptance Criteria (Gherkin Format)
```gherkin
Scenario: [Scenario Title]
  Given [Initial Context]
  When [User Action/Trigger]
  Then [Expected System Outcome]
```

## 3. Technical Constraints & Rules
List any specific business rules or technical limitations (e.g. security scopes, data validations) linked to this story.

## Optional Sections
## 4. UI/UX Wireframe Link
Link visual layouts or design components.

## Completion Checklist
- [ ] Story aligns with the INVEST framework.
- [ ] Acceptance scenarios cover positive, negative, and validation-limit cases.
- [ ] No technical execution details are mandated in the story narrative.
