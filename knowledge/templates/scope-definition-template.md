# Scope Definition Template

## Purpose
Sets clear boundaries of what is included and excluded in the project or release scope.

## Traceability
Traces back to: Project Profile, BRD. Traces forward to: Backlog, Test Scenarios.

## Metadata
- Project ID: [ID]
- Status: [Draft/Approved/Baselined]
- Last Updated: [YYYY-MM-DD]

## Required Sections
## 1. In-Scope Deliverables & Features
Detailed list of capabilities, systems, processes, and features included in the baseline scope.

## 2. Out-of-Scope Items
Explicit list of exclusions to prevent scope creep.

## 3. Boundary Assumptions
Document factors believed to limit or define these boundaries (e.g. third-party system capabilities).

## Optional Sections
## 4. Future Phase Roadmap
List items currently out-of-scope that are deferred to subsequent releases.

## Completion Checklist
- [ ] All exclusions are clear and specific (avoid generic phrases).
- [ ] In-scope items align directly with business objectives.
- [ ] Changes to this scope follow the change request process.
