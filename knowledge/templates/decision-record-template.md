# Architecture Decision Record (ADR) Template

## Purpose
Logs architectural, design, or business-logic decisions, explaining context and consequences.

## Traceability
Traces back to: BRD, Risk Register. Traces forward to: System Design, Backlog.

## Metadata
- Decision ID: [DEC-01]
- Title: [Decision Name]
- Status: [Proposed/Accepted/Superseded/Rejected]
- Owner: [Name]
- Date: [YYYY-MM-DD]

## Required Sections
## 1. Context & Problem Statement
Describe the technical/business problem, constraints, and requirements driving this decision.

## 2. Options Considered
- Option A: [Description, pros, cons]
- Option B: [Description, pros, cons]

## 3. Decision & Rationale
State the chosen option and explain why it was selected over the alternatives.

## 4. Consequences
Detail the downstream impacts of this decision (e.g. new dependencies, technical debt, training).

## Optional Sections
## 5. References
Link related specs, external resources, or discussions.

## Completion Checklist
- [ ] Consequences explicitly detail negative impacts or trade-offs.
- [ ] The status is updated when superseded by another decision.
- [ ] File naming matches repository standards.
