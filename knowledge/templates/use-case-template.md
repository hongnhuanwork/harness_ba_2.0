# Use Case Template

## Purpose
Documents detailed interaction sequences between actors and the system for a specific feature.

## Traceability
Traces back to: FRD. Traces forward to: User Stories, Test Cases.

## Metadata
- Use Case ID: [UC-01]
- Title: [Use Case Name]
- Primary Actor: [Persona/System]
- Status: [Draft/Approved]

## Required Sections
## 1. Description
Provide a summary of the business scenario.

## 2. Preconditions & Postconditions
- Preconditions: [System states required to initiate]
- Postconditions: [Final system states after success/failure]

## 3. Main Success Scenario (Happy Path)
1. [Actor Action]
2. [System Response]
3. [Actor Action] ...

## 4. Alternative & Exception Flows
- Alternative Flow A: [Condition -> Steps]
- Exception Flow B: [Error Condition -> System Message -> Exit]

## Optional Sections
## 5. UI Mockups / Wireframes
Link wireframe layouts or screen transition flow charts.

## Completion Checklist
- [ ] Preconditions and postconditions are verifiable.
- [ ] Every verification step has an associated failure exception path mapped.
- [ ] Action statements use active verbs.
