# Purpose

To check that user interfaces conform to usability goals, UX patterns, and written requirements.

# When To Use

- Wireframes or UI designs completed
- Before development handover
- Usability review tasks

# Inputs

- UI Wireframes / Mockups
- User Stories and Acceptance Criteria
- Design System guidelines

# Process

1. Map UI elements to functional requirements and Gherkin criteria.
2. Check navigation menus and buttons against Navigation Flow Specs.
3. Verify consistency with corporate branding and Design System patterns.
4. Audit for usability standards (accessibility, spacing, contrast).
5. Draft UI Feedback log.

# Output

- UI Feedback Log

# Quality Checklist

- [ ] All interactive elements map to a requirement trigger.
- [ ] Design patterns match corporate guidelines.
- [ ] Accessibility guidelines (WCAG) are met.

# Human Review Required

Yes. Product Owner and UX Designer must review feedback.
