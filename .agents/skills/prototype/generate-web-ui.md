# Purpose

To create modern, responsive, and wowed web interface designs.

# When To Use

- Web development design kickoff
- Wireframe approved
- User portal UI tasks active

# Inputs

- Approved Wireframes
- Design System guidelines
- Web layouts specifications

# Process

1. Analyze grid layout and responsive viewport sizes (desktop, tablet, mobile).
2. Apply rich aesthetics, gradients, and typography from the Design System.
3. Create high-fidelity mockups.
4. Specify hover states, animations, and transitions.
5. Output web UI specs.

# Output

- High-Fidelity Web UI Mockup (image/asset link)
- Web Layout Specification Document

# Quality Checklist

- [ ] Design meets high aesthetic standard (vibrant colors, clean modern layout).
- [ ] Layout is responsive (desktop and mobile paths mapped).
- [ ] Includes detailed element specs (contrast, sizes).

# Human Review Required

Yes. Product Owner and Frontend Tech Lead must approve.
