---

# Purpose

To visualize page layout, element placement, and user controls without style distractions.

# When To Use

- Screen design task active
- Initial UI/UX brainstorming
- User story refinement needing UI context

# Inputs

- User Stories
- Use Case specs
- Screen Transition specs

# Process

1. Identify the screen elements required (inputs, buttons, headers, tables).
2. Design layout structures using text blocks or ASCII templates.
3. If visual generator is available, request image creation for mockup.
4. Draft annotations explaining element interactions.
5. Document wireframe layout.

# Output

- Low-Fidelity Screen Wireframe Document
- Wireframe Visual Asset (image/text representation)

# Quality Checklist

- [ ] Layout contains all inputs and controls needed for the user story.
- [ ] Design is low-fidelity (no custom colors or fonts used).
- [ ] Elements conform to usability standards.

# Human Review Required

Yes. Product Owner and User Representatives must validate wireframes.
