# Purpose

To create premium, state-of-the-art mobile app interface designs using corporate design systems.

# When To Use

- Mobile design sprints active
- Wireframe approved
- UI assets requests

# Inputs

- Approved Wireframes
- Design System guidelines
- Mobile style templates

# Process

1. Review target mobile screen constraints (iOS/Android sizing).
2. Apply colors, typography, spacing, and icons from the Design System.
3. Create high-fidelity mobile mockup using image generation or Figma tools.
4. Map touch targets and responsive UI adjustments.
5. Output mobile screen UI specs.

# Output

- High-Fidelity Mobile UI Mockup (image/Figma link)
- Mobile Asset Package (icons, backgrounds)

# Quality Checklist

- [ ] Design feels premium, modern, and wows at first glance.
- [ ] Typography and spacing match mobile guidelines.
- [ ] No generic placeholders are present.

# Human Review Required

Yes. Product Owner and Lead Mobile Developer must sign off.
