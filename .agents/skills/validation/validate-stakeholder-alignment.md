---

# Purpose

To ensure consensus and alignment across different departments, preventing conflicts during delivery.

# When To Use

- Major change requests occur
- Sprint roadmap adjustments
- Conflicting feedback is detected

# Inputs

- Stakeholder register
- Expectations tracker
- Project roadmap

# Process

1. Analyze stakeholder feedback for conflicting interests.
2. Host alignment alignment sessions.
3. Present trade-off analysis (scope, time, budget).
4. Obtain written agreement on priorities.
5. Update expectation registers.

# Output

- Alignment Resolution Memo

# Quality Checklist

- [ ] All affected stakeholders are represented in alignment memos.
- [ ] Trade-offs are explicitly outlined.

# Human Review Required

Yes. All primary stakeholders must sign off.
