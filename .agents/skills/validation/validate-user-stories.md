---
name: Validate User Stories
description: Validate that user stories represent the real workflows and needs of business users.
category: validation
phase: Validation
---

# Purpose

To ensure user stories are aligned with user tasks and solve target pain points.

# When To Use

- User stories drafted
- Refinement sessions active
- UAT script preparation

# Inputs

- User Stories
- User Journey maps
- Persona definitions

# Process

1. Walk business stakeholders through user stories narratives.
2. Simulate user journeys using the stories to ensure completeness.
3. Confirm that the Gherkin criteria reflect real validation outcomes.
4. Log feedback and refine user stories.
5. Draft Story Validation Signoff.

# Output

- User Story Validation Report

# Quality Checklist

- [ ] Business stakeholders agree that the stories represent their needs.
- [ ] Workflows match user journeys.
- [ ] Errors are handled appropriately.

# Human Review Required

Yes. Product Owner and User Representatives must sign off.
