# Purpose

To write production-ready web application code, integrating logic, styling, and APIs.

# When To Use

- Development sprint active
- Approved UI Mockups and designs
- Web coding tasks assigned

# Inputs

- High-Fidelity Web UI Mockups
- API Contracts
- Design System CSS/JS tokens

# Process

1. Initialize web application workspace (HTML/CSS/JS files, or Next.js/Vite project).
2. Apply global CSS layout, typography, and utility classes.
3. Create responsive page templates and navigation.
4. Write Javascript logic for page interactions, validations, and API communication.
5. Integrate error handling and boundary scenarios.
6. Perform local browser verification.

# Output

- Web Application Source Code
- Local Dev Build configurations

# Quality Checklist

- [ ] Code compiles with no errors.
- [ ] Layout is responsive across viewport widths.
- [ ] All console logs are clean.

# Human Review Required

Yes. Frontend Architect or Lead Developer must perform a Code Review.
