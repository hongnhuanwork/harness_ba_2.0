# Purpose

To write mobile application code for iOS/Android platforms, matching UI designs and logic.

# When To Use

- Mobile development tasks active
- Approved mobile UI mockups
- Sprint coding phase active

# Inputs

- High-Fidelity Mobile UI Mockups
- API specifications
- Mobile design assets

# Process

1. Setup mobile project structure using target framework.
2. Code UI components, inputs, and screens using design assets.
3. Integrate navigation stack and routing.
4. Code api client services, state managers, and logic workflows.
5. Build and verify on target emulator/device.
6. Submit code for review.

# Output

- Mobile App Source Code
- Emulator Build files

# Quality Checklist

- [ ] Build compiles without errors.
- [ ] All navigation paths match Screen Flow specs.
- [ ] Permissions and API connectivity are validated.

# Human Review Required

Yes. Lead Mobile Developer must perform a code review.
