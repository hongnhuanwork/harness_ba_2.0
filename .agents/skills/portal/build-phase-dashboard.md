# Purpose

To track and visualize deliverables progress across the six project phases.

# When To Use

- Phase transitions scheduled
- Weekly progress updates
- PMO status reporting

# Inputs

- Project roadmap
- Output directory status
- Phase gating checklists

# Process

1. Create a visual overview of the six project phases.
2. For each phase, list mandatory deliverables and check mark their completion status.
3. Color code statuses (Not Started, In Progress, Completed, Under Review).
4. Embed links to active deliverables in the dashboard.
5. Review with the PM.

# Output

- Phase Dashboard Page (dashboard.md / portal widget)

# Quality Checklist

- [ ] Completion statuses are accurate based on files in `output/`.
- [ ] Links are functional.
- [ ] Meets styling rules.

# Human Review Required

Yes. Project Manager must approve status updates on the dashboard.
