# Purpose

To inform users and stakeholders of new system updates, scheduled downtime, and feature value.

# When To Use

- Release planning starts
- Marketing/Customer Success alignment
- Deployment schedule confirmed

# Inputs

- Release Notes
- Business value mapping
- Stakeholder roster

# Process

1. Draft email notifications for system stakeholders.
2. Draft system maintenance banners with exact downtime schedules.
3. Highlight primary user benefits and link to training materials.
4. Verify dates and times match the deployment checklist.
5. Submit draft for PR or Marketing review.

# Output

- Release Notification Drafts
- System Banner Copy

# Quality Checklist

- [ ] Downtime windows match deployment schedule exactly.
- [ ] Language is formatted for non-technical users.
- [ ] Links to guides work.

# Human Review Required

Yes. Product Owner or Client Account Lead must approve.
