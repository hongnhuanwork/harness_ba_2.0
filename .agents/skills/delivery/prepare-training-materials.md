# Purpose

To provide high-quality user training content that drives system adoption and reduces support tickets.

# When To Use

- UAT phase finishes
- System launch planning starts
- Client training request

# Inputs

- User Journeys
- Use Case specs
- System mockups / designs

# Process

1. Identify user personas and tailor training modules.
2. Draft step-by-step user guides with screenshot attachments.
3. Create quick reference cards for common workflows.
4. Write training session agendas and video walk-through scripts.
5. Review with Product Owner.

# Output

- User Training Guide
- Quick Reference Sheet

# Quality Checklist

- [ ] Content is written in user-friendly, non-technical language.
- [ ] All screenshots reflect the final user interfaces.
- [ ] Covers all primary user tasks.

# Human Review Required

No. Can be reviewed and approved by the UX Lead or Business Analyst Lead.
