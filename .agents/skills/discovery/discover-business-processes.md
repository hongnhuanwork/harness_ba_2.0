# Purpose

To understand the current operational context and map existing manual or legacy workflows before proposing new technical designs.

# When To Use

- Process improvement tasks are assigned
- Legacy system replacement planning begins
- Process flows need to be analyzed for optimization

# Inputs

- Legacy system documentation
- User interview notes and process walkthroughs
- Operational manuals

# Process

1. Identify the start and end triggers of the target business process.
2. Map sequence of tasks, tasks ownership (personas/departments), and decision nodes.
3. Identify all systems involved in the process flow.
4. Document bottlenecks, handoffs, and wait times in the current workflow.
5. Draft high-level process flowcharts using Mermaid or text representation.

# Output

- As-Is Process Description
- High-Level Process Flow Diagram (Mermaid)

# Quality Checklist

- [ ] All swimlanes (owners) are accurately identified.
- [ ] Decision points have clear binary or multi-branch conditions.
- [ ] The flow starts and ends with clearly defined triggers.

# Human Review Required

Yes. Process owners and subject matter experts must validate the accuracy of the mapped As-Is flow.
