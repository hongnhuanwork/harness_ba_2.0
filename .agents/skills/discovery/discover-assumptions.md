---
name: Discover Assumptions
description: Identify, document, and categorize assumptions made during project initiation and planning.
category: discovery
phase: Discovery
---

# Purpose

To highlight variables believed to be true but not yet verified, preventing design failures due to incorrect premises.

# When To Use

- Project charter generation
- High-level scoping discussions
- Estimation and planning sessions

# Inputs

- Project Charter
- Initial Scope drafts
- Technical proposal

# Process

1. Scan project documents for terms like 'assumed', 'expected', 'assuming', or 'suppose'.
2. List all variables (e.g., API availability, team skills, third-party timelines) taken as true.
3. Classify assumptions by category (Technical, Business, Resource, Timeline).
4. Determine the impact on project timeline/scope if the assumption is proven false.
5. Assign owner and target date for validation of each assumption.
6. Log in the Assumptions Register.

# Output

- Assumptions Register
- Assumption Validation Tracker

# Quality Checklist

- [ ] Every assumption is associated with a specific owner and validation action.
- [ ] Impact scores are assigned to evaluate the severity of assumption failure.
- [ ] Assumptions are clearly separated from validated facts.

# Human Review Required

No. Can be drafted independently but must be reviewed by the PM during planning sprints.
