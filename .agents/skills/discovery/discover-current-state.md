# Purpose

To document the current environment (As-Is state) in order to assess gaps, constraints, and integration boundaries.

# When To Use

- Legacy system modernization projects
- Gap analysis initiation
- Technical discovery phase active

# Inputs

- Legacy database schemas
- System architecture diagrams
- Existing codebase and developer documentation

# Process

1. Analyze current system architecture, network configuration, and servers.
2. Document active data schemas, tables, and flows.
3. List all active tools, platforms, licenses, and versions in use.
4. Document process throughput, user volumes, and performance baselines.
5. Compile a Current State Assessment Report.

# Output

- Current State Assessment Report
- As-Is Architecture Diagram (Mermaid)

# Quality Checklist

- [ ] All active system integrations are listed.
- [ ] Current software versions are documented.
- [ ] Known bugs or limitations of the current state are captured.

# Human Review Required

Yes. Lead Infrastructure Architect or Lead Developer must validate the accuracy of the current state document.
