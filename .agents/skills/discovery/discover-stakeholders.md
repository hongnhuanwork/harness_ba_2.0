# Purpose

To create a comprehensive Stakeholder Register and Influence Map that ensures all key parties are identified, analyzed, and engaged appropriately throughout the project lifecycle.

# When To Use

- Project kickoff or initialization phase starts
- A new project workspace is established
- Stakeholder roster updates are requested

# Inputs

- Project Charter or initial statement of work
- Organizational charts
- Meeting minutes from initial discussions

# Process

1. Scan existing PMO and project resource documents for names, titles, and department listings.
2. Identify direct users, indirect users, sponsors, decision-makers, and key external partners.
3. Map each stakeholder's influence level (High/Medium/Low) and interest level (High/Medium/Low).
4. Define expectations, primary concerns, and preferred communication channels for each persona.
5. Consolidate findings into a structured Stakeholder Register.

# Output

- Stakeholder Register (JSON/Markdown table)
- Power/Interest Grid or Influence Map

# Quality Checklist

- [ ] All sponsors and decision-makers are explicitly listed.
- [ ] Every stakeholder has a defined contact channel, interest level, and influence level.
- [ ] Roles match standard project terminology.

# Human Review Required

Yes. Product Owner or Project Manager must approve the stakeholder classification and communication plan.
