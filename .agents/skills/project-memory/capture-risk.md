# Purpose

Log project risks with Likelihood and Impact ratings, mapping out mitigation tasks.

# When To Use

- An uncertainty is identified that could delay the project, exceed the budget, or compromise security.

# Inputs

- Risk description
- Existing risks log ([risks.md](file:///d:/03_Projects/Personal/260624_HarnessAI_BA/260625_harnessBA_ver2.0/project_resources/00-project-memory/risks.md))

# Process

1. Define Risk Description, Probability, and Impact (1-5).
2. Calculate Severity (Probability x Impact).
3. Define a clear mitigation strategy and assign an owner.
4. Assign a unique ID (e.g., RSK-001).
5. Present the formatted entry to the user.

# Output

- Formatted Risk entry with mitigation

# Quality Checklist

- [ ] Verify that there is no duplicate item with same ID.
- [ ] Strictly check that the information has been approved by the user before writing.

# Human Review Required

Mandatory. Scrum Master or PM must approve risk classifications before writing to risks.md.
