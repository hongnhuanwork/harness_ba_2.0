# Purpose

Systematically log lessons learned to improve future sprint cycles and workflows.

# When To Use

- A project milestone is completed, or a process issue is resolved.

# Inputs

- Retrospective notes
- Existing lessons log ([lessons-learned.md](file:///d:/03_Projects/Personal/260624_HarnessAI_BA/260625_harnessBA_ver2.0/project_resources/00-project-memory/lessons-learned.md))

# Process

1. Extract the Situation, Observation, Core Lesson, and Recommendation.
2. Assign a unique ID (e.g., LSN-001).
3. Format as a markdown table row.
4. Present draft to the user.

# Output

- Formatted Lesson Learned entry

# Quality Checklist

- [ ] Verify that there is no duplicate item with same ID.
- [ ] Strictly check that the information has been approved by the user before writing.

# Human Review Required

Mandatory. Business Analyst Lead must approve the lesson format before writing to lessons-learned.md.
