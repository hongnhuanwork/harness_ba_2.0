# Purpose

To translate functional requirements into logical architectural blueprints and technical designs.

# When To Use

- Requirements baseline is signed off
- Architectural design sessions begin
- Prototyping phase starts

# Inputs

- Functional Requirements Specification (FRS)
- Technical constraints
- Current State Architecture maps

# Process

1. Define system modules, components, and tiers.
2. Design logical architecture flows showing system components and interfaces.
3. Define deployment models, tech stacks, and environment structures.
4. Review logical design with tech leads to confirm alignment with NFRs.
5. Draft Solution Design Document.

# Output

- Solution Design Document (SDD)
- System Context Diagram (Mermaid)

# Quality Checklist

- [ ] The design addresses all functional and non-functional requirements.
- [ ] Tech stack matches defined constraints.
- [ ] Interfaces and interfaces protocols are specified.

# Human Review Required

Yes. Lead Solution Architect and Product Owner must sign off.
