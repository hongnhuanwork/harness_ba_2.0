# Purpose

To structure the application menu architecture and path logic, ensuring intuitive user navigation.

# When To Use

- UX/UI design kickoff
- Site map creation tasks
- User Journey mapping completed

# Inputs

- User Journeys
- Persona definitions
- Scope list

# Process

1. Group application screens into logical categories and sub-menus.
2. Define navigation paths between sections (e.g., deep-linking, tab bars).
3. Establish user permission roles and menu visibility rules.
4. Draw high-level Site Map and navigation paths.
5. Consolidate in a Navigation Flow Spec.

# Output

- Information Architecture / Site Map
- User Navigation Path Specification

# Quality Checklist

- [ ] Click depth for core tasks does not exceed 3 levels.
- [ ] Role-based menu changes are mapped.
- [ ] Navigation paths are circular, preventing dead-ends.

# Human Review Required

No. Can be approved by UX/UI Lead or Product Designer.
