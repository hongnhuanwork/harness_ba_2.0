# Purpose

To create streamlined, optimized, and automated business workflows that replace legacy processes.

# When To Use

- Process analysis is completed
- System modernization planning begins
- Workflow automation scoping is active

# Inputs

- As-Is Process Descriptions
- Process Optimization recommendations
- Functional specifications

# Process

1. Define the optimized target workflow triggers, steps, and outcomes.
2. Eliminate waste steps, manual handoffs, and duplicate validations identified in the current state.
3. Integrate automated system tasks into the process workflow.
4. Create process flowcharts using BPMN or Mermaid notation.
5. Document the To-Be process step-by-step.

# Output

- To-Be Process Specification Document
- To-Be Process Flow Diagram (Mermaid/BPMN)

# Quality Checklist

- [ ] The target flow is optimized, showing metrics-backed reductions in steps/time.
- [ ] Manual steps are clearly distinguished from automated steps.
- [ ] Exceptions and error-handling steps are mapped.

# Human Review Required

Yes. Business Unit Lead or Process Owner must approve To-Be workflow.
