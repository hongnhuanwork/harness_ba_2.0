# Purpose

To define report contents, targets, layouts, refresh frequencies, and access permissions.

# When To Use

- Analytics and BI design starts
- KPI tracking planning
- Business reporting meetings

# Inputs

- KPI Definition Register
- Stakeholder Register
- Business goals list

# Process

1. Identify target audience (Executive, Managerial, Operational) for each report.
2. Define required data fields, metrics, filters, and grouping variables.
3. Document visualization preferences (Charts, tables, dashboards).
4. Define report schedule (Daily, Weekly, Monthly) or real-time query needs.
5. Log reporting requirements in a BI specifications document.

# Output

- Reporting Requirements Document
- Report Mockup / Layout Blueprint

# Quality Checklist

- [ ] Metrics match definitions in the KPI register.
- [ ] Data filters and permissions are specified.
- [ ] Data sources for every field are identified.

# Human Review Required

Yes. Business Analyst Lead or PO must approve reporting layouts and fields.
