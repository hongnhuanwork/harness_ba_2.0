# Purpose

To understand end-user needs, tasks, and pain points to design interfaces and features that deliver maximum user value.

# When To Use

- User research active
- UX/UI design kickoff
- Persona building sessions

# Inputs

- User interview notes
- UX research summaries
- Customer support tickets

# Process

1. Extract user goals, tasks, and frustrations from inputs.
2. Define user personas with target attributes, goals, and technical literacy.
3. Map key user tasks to workflow steps.
4. Translate user needs into capabilities ('User Needs statements').
5. Log user needs and link them to user stories.

# Output

- User Needs Registry
- User Persona Definitions

# Quality Checklist

- [ ] Needs are stated from the user's perspective, not the system's.
- [ ] Personas are based on actual user data/interviews.
- [ ] Target workflows contain clear actions and triggers.

# Human Review Required

No. Can be reviewed and refined by the UX Lead or Product Designer.
