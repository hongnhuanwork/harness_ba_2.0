# Purpose

To align stakeholder perceptions and project scope, ensuring that critical requirements and expectations are managed.

# When To Use

- Steering committee meetings
- Project scope refinements
- Stakeholder misalignment is detected

# Inputs

- Stakeholder Register
- Project SOW / Charter
- Meeting transcripts

# Process

1. Identify subjective success criteria from key stakeholders (e.g., 'must look modern').
2. Translate subjective expectations into objective, measurable requirements.
3. Map conflicts between stakeholder expectations (e.g., scope vs. budget).
4. Propose alignment resolutions and document compromises.
5. Record aligned expectations in the expectations tracker.

# Output

- Stakeholder Expectations Tracker
- Expectation Alignment Agreement

# Quality Checklist

- [ ] Expectations are translated to measurable targets.
- [ ] Conflicts are explicitly documented and resolved.
- [ ] Updates are communicated back to stakeholders.

# Human Review Required

Yes. Product Owner or Project Manager must approve the alignment resolutions.
