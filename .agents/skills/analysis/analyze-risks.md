# Purpose

To perform qualitative and quantitative risk analysis, updating risk matrices with actionable mitigation plans.

# When To Use

- Risk registers need update
- Risk events occur
- Project phase transitions

# Inputs

- Initial Risk Register
- Feasibility reports
- Technical constraints

# Process

1. Re-assess existing risks for changes in likelihood or impact.
2. Brainstorm mitigation strategies (Avoid, Mitigate, Transfer, Accept).
3. Assign risk owners and action items.
4. Establish contingency budgets or backup timelines for critical items.
5. Update the Risk Register.

# Output

- Updated Risk Register
- Mitigation Action Plan

# Quality Checklist

- [ ] All risks have a named owner.
- [ ] Mitigation actions are concrete and scheduled.
- [ ] Risk scores are recalculated post-mitigation planning.

# Human Review Required

Yes. Project Manager and Tech Lead must approve mitigation plans.
