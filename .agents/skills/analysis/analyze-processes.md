# Purpose

To evaluate process flows for inefficiencies using Lean/Six Sigma or standard business analysis mapping tools.

# When To Use

- As-Is Process discovery completes
- Business process re-engineering projects
- Performance optimization reviews

# Inputs

- As-Is Process Descriptions
- Process performance metrics (processing time, volume)
- User pain points documentation

# Process

1. Map process cycle times, wait times, and total throughput times.
2. Identify waste categories (Overproduction, Waiting, Transport, Excess Processing, Motion, Defects).
3. Pinpoint handoff bottlenecks and system integrations gaps.
4. Recommend process automation or restructuring opportunities.
5. Draft Process Optimization Report.

# Output

- Process Optimization Report
- Value Stream Map

# Quality Checklist

- [ ] Bottlenecks are quantified with metric data (time or cost).
- [ ] Root causes of delays are clearly documented.
- [ ] Recommendations align with technology constraints.

# Human Review Required

Yes. Process Owners must review and approve optimization paths.
