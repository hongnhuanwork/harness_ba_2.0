category: analysis

---

# Purpose

To identify system, process, data, and skill gaps that must be addressed to achieve the desired solution state.

# When To Use

- Current State Discovery is complete
- To-Be process designs are drafted
- Migration planning active

# Inputs

- Current State Assessment Report
- Proposed To-Be process models
- Target business goals

# Process

1. Compare current systems/processes against target capabilities.
2. Identify missing functionality, data objects, integrations, and operational steps.
3. Categorize gaps (System, Data, Process, Resource).
4. Estimate impact of each gap on project goals.
5. Propose high-level solutions or projects to bridge the gaps.

# Output

- Gap Analysis Report
- Gap Resolution Roadmap

# Quality Checklist

- [ ] All gaps are matched with a resolution recommendation.
- [ ] Gaps are ranked by criticality.
- [ ] Impact descriptions are concrete and objective.

# Human Review Required

Yes. Solution Architect and Business Sponsor must review and approve.
