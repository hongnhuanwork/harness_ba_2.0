# Purpose

To ensure designed features and data policies strictly comply with governing laws and regulatory standards.

# When To Use

- Requirements phase completes
- Before design finalization
- Compliance audits active

# Inputs

- Compliance Requirements Specification
- Draft System Designs
- Security standards

# Process

1. Map proposed database tables, encryption standards, and workflows against compliance requirements.
2. Audit consent forms, opt-out mechanisms, and user profile fields.
3. Identify potential compliance gaps or regulatory risks.
4. Define required audit logs, access scopes, and reporting needs.
5. Submit Compliance Compliance Report.

# Output

- Compliance Audit Report
- Risk Remediation Plan

# Quality Checklist

- [ ] Regulatory guidelines are successfully mapped to specific system actions.
- [ ] Privacy by Design features are clearly identified.
- [ ] Any gaps are reported directly to the Security Officer.

# Human Review Required

Yes. Legal/Compliance Officer and DPO (Data Protection Officer) must review.
