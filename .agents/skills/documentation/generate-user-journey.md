# Purpose

To document the end-to-end user experience, helping team members align on user needs and emotional context.

# When To Use

- User research active
- UX design planning
- Feature enhancement scoping

# Inputs

- User Persona definitions
- Elicited user needs
- Customer feedback data

# Process

1. Define the scope and goal of the user journey.
2. Map journey steps (e.g. Discovery, Signup, Onboarding, Engagement).
3. For each step, detail user actions, touchpoints, thoughts, and emotions.
4. Highlight pain points and opportunities at each step.
5. Create a clean markdown table representing the Journey Map.

# Output

- User Journey Map Document

# Quality Checklist

- [ ] Map is based on a defined User Persona.
- [ ] All touchpoints are real system screens/features.
- [ ] Pain points align with the Pain Points Register.

# Human Review Required

No. Can be validated by UX Lead or Product Designer.
