# Purpose

To document discussion topics, decisions, actions, and owners from workshops or alignments.

# When To Use

- Workshops or client alignments complete
- Sprint planning or reviews finish
- Alignment issues need documentation

# Inputs

- Raw meeting notes / recordings
- Meeting agenda and attendees list

# Process

1. Retrieve Meeting Minutes template from `knowledge/templates/`.
2. Document metadata (Date, Time, Attendees, Absentees, Objectives).
3. Summarize discussion topics and key arguments.
4. List all decisions made, referencing ADR IDs if applicable.
5. Create a clean action items table showing: Task, Owner, Due Date.
6. Distribute to attendees.

# Output

- Meeting Minutes Document

# Quality Checklist

- [ ] All decisions made are explicitly recorded.
- [ ] Action items have designated owners and realistic due dates.
- [ ] Formatting is clean and consistent.

# Human Review Required

No. Can be reviewed and approved by the Business Analyst Lead.
