# Purpose

To draft business-readable test scenarios that enable clients/stakeholders to formally accept the system.

# When To Use

- UAT planning phase starts
- Client demo preparation
- System deployment preparation

# Inputs

- User Stories
- Acceptance Criteria
- User Journey maps

# Process

1. Retrieve UAT template from `knowledge/templates/`.
2. Translate technical QA test steps into business-friendly user workflows.
3. Define realistic test scenarios (e.g. 'Book a ticket from start to finish').
4. Provide clear success criteria and columns for pass/fail/notes.
5. Package instructions with logging templates.
6. Submit to Product Owner.

# Output

- User Acceptance Testing (UAT) Scripts

# Quality Checklist

- [ ] Language is free of development jargon.
- [ ] Scenarios represent complete business tasks.
- [ ] Includes sections for client signature and date.

# Human Review Required

Yes. Product Owner or Client Lead must sign off on UAT scripts.
