# Purpose

To define the project's high-level objectives, sponsors, budget, timelines, scope, and authority limits, initiating the project.

# When To Use

- Project initialization starts
- Sponsorship finalized
- Establishment of project workspace

# Inputs

- Statement of Work (SOW)
- Client proposal
- Business goals drafts

# Process

1. Retrieve Project Charter template from `knowledge/templates/`.
2. Document Project Title, Sponsors, PM, and Client contact.
3. Define High-Level Scope, Business Objectives, and Success Metrics.
4. Outline key milestones, timelines, and budget limits.
5. List principal risks, assumptions, and constraints.
6. Document project organization and authority boundaries.
7. Submit for signatures.

# Output

- Project Charter

# Quality Checklist

- [ ] All fields in the Charter template are fully populated.
- [ ] Timeline milestones are mapped to specific quarters/dates.
- [ ] Sponsor details are correct.

# Human Review Required

Yes. Executive Sponsor and Project Manager must sign off.
