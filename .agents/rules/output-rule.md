# Output Rule

## Purpose
This rule governs the storage, organization, naming conventions, and delivery of artifacts generated by the AI Business Analyst Agent. It ensures that the output directory remains structured, version-controlled, and easily navigable for both human stakeholders and downstream development/testing processes.

## Principles
1. **Order and Organization**: Maintain a clean, logical file structure that reflects the current state of the business analysis lifecycle.
2. **Standardized Naming**: Enforce clear, readable, and consistent filenames across all folders.
3. **Traceable Versioning**: Systematically track changes to outputs to prevent data loss or conflicting definitions.
4. **Lifecycle Alignment**: Store files within the directory corresponding to the specific phase of analysis in which they were created.

## Rules
1. **Directory Structure Guidelines**:
   - All generated business analysis artifacts must be saved under the appropriate sub-directory in the `output/` folder:
     - `output/phase-1-discovery/`
     - `output/phase-2-elicitation/`
     - `output/phase-3-analysis/`
     - `output/phase-4-design/`
     - `output/phase-5-validation/`
     - `output/phase-6-delivery/`
2. **File Naming Conventions**:
   - Format filenames as: `<phase_code>-<artifact_type>-<name>-v<version>.<extension>`
   - Use lowercase characters and hyphens for word separation (e.g., `p3-brd-payment-gateway-v1.0.md`).
   - Phase Codes: `p1` (Discovery), `p2` (Elicitation), `p3` (Analysis), `p4` (Design), `p5` (Validation), `p6` (Delivery).
3. **Master Output Log**:
   - Maintain a master catalog file at `output/README.md` containing a table of all active artifacts, their version, status, description, and link.
4. **Versioning and Overwriting**:
   - Increment major versions (`v1.0 -> v2.0`) for structural changes or significant requirement modifications approved by the user.
   - Increment minor versions (`v1.0 -> v1.1`) for formatting, spelling, or styling corrections.
   - Do not overwrite existing major version files directly; instead, create a new file or archive the old version in an `archive/` folder if it has been formally baseline-approved.

## Exceptions
- **Temporary Scripts**: Python scripts or shell scripts used to automate data extraction or validation are stored in the `/scratch/` directory and are exempt from naming/path restrictions.
- **System Logs**: Operational logs, error dumps, and command outputs do not belong in the `output/` directory.

## Success Criteria
- **Strict Path Compliance**: 100% of generated BA artifacts are located inside one of the six phase subfolders under `output/`.
- **Naming Pattern Match**: All filenames conform exactly to the `<phase_code>-<artifact_type>-<name>-v<version>.<extension>` pattern.
- **Index Accuracy**: The master index `output/README.md` contains up-to-date and accurate links to all files in the output folders.
